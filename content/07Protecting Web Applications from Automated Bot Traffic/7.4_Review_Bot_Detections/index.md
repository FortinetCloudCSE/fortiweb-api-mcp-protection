## Exercise 7.4 – Review Bot Detections

### Objective

Review events generated by the legitimate and automated scenarios, identify the responsible bot-detection engine, and confirm the action FortiWeb applied.

### Step 1 – Open the Attack Log

Navigate to:

**Log & Report → Log Access → Attack**

Filter for the Juice Shop host or server policy.

![Attack Log filtered for Juice Shop bot events — add screenshot](bot-attack-log-overview.png)

### Step 2 – Identify Bot Events

Depending on model state and generated traffic, you may see:

* Bot Detection
* Malicious Bot
* Vulnerability Scanner
* Crawler Detection
* Threshold violation
* Browser validation
* ML-Based Bot Detection

![Bot detection categories in Attack Log — add screenshot](bot-detection-categories.png)

{{% notice note %}}
ML-Based Bot Detection may require sufficient samples and model-building time. Not every event category is guaranteed to appear during a short lab run.
{{% /notice %}}

### Step 3 – Inspect Event Details

Open several entries and review:

* Source address and user agent
* Requested URL
* Main type and subtype
* Detection policy
* Triggered threshold or bot classification
* Action and severity
* Timestamp

![Detailed Bot Mitigation event — add screenshot](bot-event-detail.png)

### Step 4 – Compare Legitimate and Automated Traffic

Use the Traffic and Attack Logs to compare normal browsing with bot traffic. Determine whether legitimate requests were unaffected and whether automated requests triggered the expected controls.

### Reflection Questions

1. Which detection method generated the most events?
2. Which controls denied traffic and which only alerted?
3. Did any legitimate browsing request appear suspicious?
4. What threshold would you tune first if false positives occurred?
5. Why should known-good bot bypass rules be narrowly scoped?

### Chapter Summary

You configured biometric, threshold, known-bot, and ML-based detection; applied the policies to Juice Shop; generated normal and automated traffic; and reviewed the resulting events. The layered approach improves visibility and coverage against both known and previously unseen automation.
