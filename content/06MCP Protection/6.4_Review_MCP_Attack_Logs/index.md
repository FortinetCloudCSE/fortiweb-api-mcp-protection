## Exercise 6.4 – Review MCP Attack Logs

### Objective

Analyze the events generated by the MCP Attack Campaign and identify which FortiWeb protection detected each request.

### Step 1 – Open the Attack Log

Navigate to:

**Log & Report → Log Access → Attack**

Filter for `mcp.fortiweblab.local` or the MCP server policy used in your lab.

![Attack Log filtered for MCP events — add screenshot](mcp-attack-log-overview.png)

### Step 2 – Review Detection Categories

Look for events associated with:

#### Signature Detection

Traditional attacks such as SQL Injection, XSS, Command Injection, and Directory Traversal may be detected inside MCP tool parameters.

#### MCP JSON Schema Validation

Malformed JSON-RPC messages or invalid field types may be rejected before reaching the MCP server.

#### Prompt Poisoning Protection

Prompts or arguments intended to override instructions, access unauthorized tools, or retrieve sensitive information may be classified as prompt-poisoning attempts.

![MCP log entries showing multiple detection categories — add screenshot](mcp-detection-categories.png)

### Step 3 – Inspect Event Details

Open several events and review:

* Source address
* Threat type and score
* Server policy
* Requested URL and host
* User agent
* JA4 TLS fingerprint, when available
* Matched protection
* Action taken

![Detailed MCP attack event — add screenshot](mcp-attack-log-detail.png)

### Step 4 – Compare With Legitimate MCP Traffic

Compare an attack entry with a successful transaction from Exercise 6.1. Note the differences in action, response code, detection reason, and message structure.

### Reflection Questions

1. Which attacks were detected by signatures?
2. Which events were classified as MCP schema violations?
3. Did prompt-protection events identify unauthorized intent, malicious tool arguments, or both?
4. Which log field confirms whether the request was denied?
5. Why must MCP security include authorization and least privilege in addition to protocol inspection?

### Chapter Summary

You generated legitimate MCP traffic, configured MCP-aware security, launched an attack campaign, and reviewed FortiWeb detections. The exercises demonstrate how FortiWeb extends application protection to AI tool interactions using signatures, prompt protection, and JSON schema validation.
